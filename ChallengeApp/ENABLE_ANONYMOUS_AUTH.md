# Включение анонимной авторизации в Supabase

## Проблема

Вы видите ошибку:
```
api(message: "Anonymous sign-ins are disabled", errorCode: Auth.ErrorCode(rawValue: "anonymous_provider_disabled")
```

Это означает, что анонимная авторизация отключена в настройках вашего Supabase проекта.

## Решение: Включить анонимную авторизацию

### Шаг 1: Откройте Supabase Dashboard

1. Перейдите на https://supabase.com/dashboard
2. Выберите ваш проект: `qvyxkbeafgarcjjppttd`

### Шаг 2: Перейдите в настройки Authentication

1. В левом меню нажмите **Authentication**
2. Перейдите на вкладку **Providers**

### Шаг 3: Включите Anonymous Provider

1. Найдите **Anonymous** в списке провайдеров
2. Переключите переключатель в положение **ON** (включено)
3. Нажмите **Save**

### Шаг 4: Проверьте настройки

После включения анонимная авторизация будет работать автоматически. Приложение сможет:
- Автоматически авторизовывать пользователей анонимно
- Восстанавливать сессии при запуске
- Вступать в челленджи без явного входа

---

## Альтернатива: Использовать Email/Password авторизацию

Если вы не хотите использовать анонимную авторизацию, можно:

1. **Включить Email/Password провайдер** в Supabase Dashboard
2. **Использовать реальную авторизацию** через `AuthView`
3. **Убрать автоматическую анонимную авторизацию** из кода

---

## После включения

После включения анонимной авторизации:

1. **Перезапустите приложение**
2. **Попробуйте вступить в челлендж** - ошибка 401 должна исчезнуть
3. **Проверьте логи** - должно появиться сообщение об успешной авторизации

---

## Проверка

Чтобы проверить, что анонимная авторизация включена:

1. Откройте Supabase Dashboard → Authentication → Providers
2. Убедитесь, что **Anonymous** включен (зеленый переключатель)
3. Попробуйте авторизоваться в приложении - должно работать без ошибок
